Module 8 — Data Modeling Part I: Fundamentals
Date: March 2026

================================================================
WHICH MODELS DID I CHOOSE FIRST AND WHY?
================================================================

In this module I transitioned from data wrangling and feature
engineering to the modeling phase of the project. The goal was
to train baseline machine learning models that can predict
player performance using the engineered feature datasets.

I developed two separate regression models corresponding to
the two analytical objectives of the project.

The first model predicts batting performance. The target
variable is runs scored by a batter in a match. Because runs
are a continuous numeric variable this problem is formulated
as a regression task. I selected Linear Regression as the
first model because:
- It is the simplest and most interpretable model available
- It establishes a performance baseline all other models must beat
- It is computationally fast on a dataset of 16,515 rows
- It makes the fewest assumptions about the data structure
- It is appropriate for continuous prediction targets

The second model predicts bowling performance. The target
variable is economy rate — the average runs conceded per over.
Economy rate is also a continuous variable making this another
regression problem. Linear Regression was again used as the
baseline model for consistency and interpretability.

================================================================
WHAT ASSUMPTIONS DOES THE MODEL MAKE?
================================================================

Linear Regression assumes:

1. A linear relationship between features and the target variable.
   This assumption is likely violated for batting since player
   performance has nonlinear interactions with venue and stage.

2. Independence of observations. Each match row is treated
   as independent which is a reasonable approximation for
   cross-sectional match data.

3. Homoscedasticity — constant variance of errors across all
   predictions. Batting runs have high variance at the upper
   end which may violate this assumption.

4. No severe multicollinearity between input features.
   Some features like balls and strike_rate are correlated
   which may affect coefficient stability.

These assumptions motivated the later evaluation of Ridge
and Lasso regression which relax the multicollinearity
concern through regularisation.

================================================================
WHAT SPLIT DID I CHOOSE AND WHY?
================================================================

I implemented two split strategies:

Strategy 1 — Random 70/15/15 split for initial baseline:
- 70% training : 11,560 rows
- 15% validation: 2,477 rows
- 15% test      : 2,478 rows

This was used for early exploration to quickly establish
baseline performance numbers.

Strategy 2 — Time-based chronological split for all final models:
- Train      : seasons 2008–2021 (13,071 rows)
- Validation : season 2022      (1,158 rows)
- Test        : seasons 2023–2024 (2,286 rows)

The time-based split is more appropriate for sports data because:
- It mirrors real-world usage — we always predict future matches
  from past history only
- A random split would allow future seasons to appear in training,
  giving the model information it cannot have at prediction time
- It tests genuine generalisation across time rather than just
  across randomly sampled rows

The time-based split was used for all final model comparisons
and reported results.

================================================================
BASELINE MODEL RESULTS
================================================================

Batting — Linear Regression (Time-Based Split):
  Train R²: 0.9244 | MAE: 3.97 | RMSE: 5.79
  Val R²  : 0.9232 | MAE: 4.25 | RMSE: 5.96
  Test R² : 0.8931 | MAE: 5.22 | RMSE: 7.45

Bowling — Linear Regression (Time-Based Split):
  Train R²: 0.8896 | MAE: 0.54 | RMSE: 0.98
  Val R²  : 0.7696 | MAE: 0.89 | RMSE: 1.50
  Test R² : 0.7547 | MAE: 1.17 | RMSE: 1.64

The batting model shows Train and Val R² very close together
(0.9244 vs 0.9232) indicating good generalisation with no
severe overfitting. The test R² drop to 0.8931 reflects the
model being evaluated on the most recent unseen seasons.

The bowling model shows a larger drop from Train to Val
(0.8896 → 0.7696) indicating that bowling performance is
harder to model and more variable across seasons.

================================================================
DIAGNOSING UNDERFITTING VS OVERFITTING
================================================================

Batting model diagnosis:
  Train R²: 0.9244 | Test R²: 0.8931 | Gap: 0.0313
  Gap is small → model is generalising well
  Conclusion: slight underfitting remains — nonlinear
  models may capture additional patterns

Bowling model diagnosis:
  Train R²: 0.8896 | Test R²: 0.7547 | Gap: 0.1349
  Gap is moderate → mild overfitting on training seasons
  Conclusion: regularisation may help by preventing the
  model from overfitting to specific bowler-venue combinations

================================================================
BIAS-VARIANCE TRADEOFF
================================================================

The bias-variance tradeoff describes the tension between two
sources of prediction error:

Bias — error from overly simplistic assumptions.
A high-bias model underfits and misses real patterns.
Linear Regression has moderate bias here because batting
performance has nonlinear relationships it cannot capture.

Variance — error from sensitivity to training data.
A high-variance model overfits and memorises training data
but fails on new data. Later, Gradient Boosting showed
near-perfect training scores (R² > 0.99) demonstrating
high variance from in-match leakage features.

The Goldilocks principle: we want a model complex enough
to capture real patterns but simple enough to generalise.
Random Forest with max_depth=5 strikes this balance:
  Train R²: 0.96 | Test R²: 0.95 | Gap: 0.01

================================================================
WHAT RISKS REMAIN?
================================================================

1. Data leakage risk:
   Features like balls, strike_rate and runs_conceded are
   in-match statistics. They describe outcomes of the match
   being predicted. The high R² scores may be partly inflated
   by this leakage. This was investigated and documented as
   a project limitation.

2. Temporal drift risk:
   Player form changes over time. A model trained on 2008-2021
   data may not fully capture 2024 playing styles and conditions.

3. Missing context risk:
   Pitch conditions, weather, toss result and opposition
   lineup strength are not in the dataset. These factors
   significantly influence match outcomes in cricket.
