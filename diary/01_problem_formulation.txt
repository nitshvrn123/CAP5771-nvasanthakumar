# DIARY ENTRY: PROBLEM FORMULATION

## Project Objective

In professional cricket, performance is commonly summarized using aggregate statistics such as batting average, strike rate, economy rate, or wickets. While these metrics describe overall ability, they do not explicitly capture how performance changes under high-stakes situations.

In the Indian Premier League (IPL), competitive pressure varies by tournament stage. League matches allow recovery across a long season, while playoffs and finals introduce elimination risk, heightened scrutiny, and increased competitive intensity.

This project investigates whether later tournament stages (playoffs and finals) are associated with higher performance risk and variability for players compared to league matches, and builds regression models to predict future player performance under similar contextual conditions.

---

## Operational Definition of Pressure

Competitive pressure is operationalized using match stage:

- League  
- Playoffs  
- Final  

League matches serve as the baseline. Playoffs and finals represent progressively higher competitive pressure.

Rather than collapsing all non-league matches into a single group, league, playoffs, and finals are treated as distinct categories. This preserves tournament structure and enables fine-grained comparison across increasing pressure levels.

---

## Analytical Focus

This project has two connected components:

### 1️⃣ Descriptive Analysis — Performance Risk & Variability

The first objective is to quantify how pressure shifts the distribution of individual performance for both batters and bowlers.

### Batting Outcomes
- Runs scored per match  
- Probability of a low score (runs < 10)  
- Variance of runs  
- Stage-specific shifts in average performance  
- Batting Pressure Impact Index  

### Bowling Outcomes
- Wickets per match  
- Economy rate  
- Variance of economy  
- Probability of a high-cost spell  
- Bowling Pressure Impact Index  

The focus is not on predicting match winners, but on understanding individual performance risk under varying competitive conditions.

---

### 2️⃣ Predictive Modeling — Player-Level Regression

The second objective is to build supervised regression models that predict player performance in a future match under similar contextual circumstances.

Two regression tasks are defined:

#### Batting Model

**Goal:** Predict runs scored in a match.

**Target Variable:**
- Runs scored  

**Features:**
- Historical average runs  
- Run variance  
- Low-score probability  
- Match stage  
- Season  
- Venue  
- Date-derived features (month, weekday)

---

#### Bowling Model

**Goal:** Predict economy rate in a match.

**Target Variable:**
- Economy rate  

**Features:**
- Historical economy rate  
- Wicket rate  
- Dot-ball rate  
- Match stage  
- Season  
- Venue  
- Date-derived features

All models are trained using only historical matches prior to prediction to prevent data leakage.

---

## Unit of Analysis

The dataset is structured at the player–match level.  
Each observation represents a single player’s performance in a specific IPL match.

---

## Time Horizon

The dataset covers IPL seasons from 2008 to 2024.

All descriptive analysis and predictive modeling are performed using structured historical match data within this period.

---

## Summary

This project integrates:

- Structured data acquisition  
- Relational database design  
- Stage-aware performance risk analysis  
- Feature engineering for regression  
- Player-level predictive modeling  

The result is a reproducible analytical framework for understanding and predicting player performance under competitive pressure in professional cricket.
