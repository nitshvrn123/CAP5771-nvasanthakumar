DIARY ENTRY: DATA ACQUISITION (SOURCES AND RELEVANCE)

Project Objective

The goal of this project is to analyze how competitive pressure affects performance variability and risk in professional cricket. Rather than focusing on predicting average outcomes, the analysis emphasizes uncertainty-related performance measures such as low-score frequency, variability, and inconsistency across different match contexts.

Pressure is operationalized using tournament structure and match stage (league, playoffs, finals), which provides a clear, objective, and reproducible proxy for competitive pressure.

⸻

Primary Data Source

Cricsheet
https://cricsheet.org

Cricsheet is a publicly available and widely used repository of structured ball-by-ball cricket data. The dataset covers Indian Premier League (IPL) matches from 2008 to 2024, making it suitable for longitudinal and comparative analysis.

⸻

Data Description

The acquired dataset includes detailed ball-by-ball information that was aggregated into player–match level records. Key components include:

Match-Level Data
	•	Match ID
	•	Date and season
	•	Venue
	•	Match type
	•	Tournament stage (league, playoffs, final)

Player-Level Performance Data
	•	Runs scored
	•	Balls faced
	•	Strike rate
	•	Dismissal indicator
	•	Match participation

Both batting and bowling participants are retained in the dataset, enabling analysis across player roles and supporting future extensions of the project.

⸻

Derived Variables

To support the study of performance risk, additional variables were derived from the raw data:
	•	Low-score indicator (e.g., runs < 10)
	•	Player-level performance variability
	•	Match-level aggregates such as average runs and low-score rates

These derived features allow the project to quantify performance uncertainty rather than relying solely on average outcomes.

⸻

Data Acquisition Process
	•	Raw IPL match data was downloaded from Cricsheet in YAML format
	•	Python scripts were used to parse ball-by-ball records
	•	Parsed data was aggregated into player–match level CSV files
	•	The processed datasets were stored in a structured format suitable for database integration and statistical analysis

⸻

Relevance to the Project

This dataset provides objective, reproducible ground-truth measurements of player performance under varying competitive conditions. It directly supports:
	•	Comparison of performance across match stages
	•	Analysis of performance risk under high-pressure matches
	•	Player-level and match-level exploratory analysis

By relying solely on structured cricket data and tournament context, the project avoids speculative assumptions and maintains a clear analytical scope.

⸻

Data Accessibility and Reproducibility
	•	All processed datasets are stored locally in CSV format
	•	Data can be regenerated from original Cricsheet sources using documented scripts
	•	A unified SQLite database integrates all tables for reproducibility
	•	File naming conventions and repository structure ensure transparency

⸻

Summary

The Cricsheet IPL dataset provides a high-quality, structured foundation for analyzing performance variability under competitive pressure. Its completeness, temporal coverage, and detailed granularity make it well-suited for studying how match context influences player performance risk in professional cricket.
