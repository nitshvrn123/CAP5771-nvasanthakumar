DIARY ENTRY: DATA ACQUISITION (SOURCES AND RELEVANCE)

Project Context

The goal of this project is to analyze how competitive pressure affects performance risk and variability in IPL players. Pressure is defined using tournament stage:
	•	league
	•	playoffs
	•	final

The dataset spans IPL seasons from 2008 to 2024, allowing long-term comparison across match stages.

⸻

Primary Data Source

Cricsheet
https://cricsheet.org

Cricsheet provides structured ball-by-ball IPL match data in YAML format. It was selected because it is:
	•	Public and reproducible
	•	Complete for IPL seasons
	•	Structured and suitable for aggregation

⸻

Data Description

The raw ball-by-ball data was aggregated to player–match level records.

Match-Level Data
	•	Match ID
	•	Date and season
	•	Venue
	•	Match type
	•	Stage (league / playoffs / final)

Player-Level Data

Batting:
	•	Runs
	•	Balls faced
	•	Strike rate

Bowling:
	•	Overs
	•	Runs conceded
	•	Wickets
	•	Economy rate

⸻

Derived Variables

To study performance risk, I created:
	•	Low-score indicator (runs < 10)
	•	Performance variance
	•	Stage-based averages
	•	Pressure Impact Indices

These features allow analysis of uncertainty rather than just average performance.

⸻

Data Processing Pipeline
	1.	Downloaded raw IPL YAML files from Cricsheet
	2.	Parsed using Python
	3.	Aggregated to player–match level
	4.	Stored in CSV files
	5.	Integrated into a SQLite database (ipl_cricket_only.db)

Relevance to the Project

This dataset provides objective, reproducible ground-truth measurements of player performance under varying competitive conditions. It directly supports:
	•	Comparison of performance across match stages
	•	Analysis of performance risk under high-pressure matches
	•	Player-level and match-level exploratory analysis

By relying solely on structured cricket data and tournament context, the project avoids speculative assumptions and maintains a clear analytical scope.

⸻

Data Accessibility and Reproducibility
	•	All processed datasets are stored locally in CSV format
	•	Data can be regenerated from original Cricsheet sources using documented scripts
	•	A unified SQLite database integrates all tables for reproducibility
	•	File naming conventions and repository structure ensure transparency

⸻

Summary

The Cricsheet IPL dataset provides a high-quality, structured foundation for analyzing performance variability under competitive pressure. Its completeness, temporal coverage, and detailed granularity make it well-suited for studying how match context influences player performance risk in professional cricket.
