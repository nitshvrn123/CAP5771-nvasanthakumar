Module 10 — Data Visualization: Communicating Results
Date: March 2026

================================================================
REFLECTION: What information is most important to highlight?
================================================================

At this stage of the project the most important information to
communicate is:

1. How well the models predict batting and bowling performance
   — the audience needs to know if the models are trustworthy
   before acting on any predictions.

2. Which features drive predictions most strongly
   — helps cricket analysts understand what historical factors
   matter most when predicting player performance.

3. How predictions change based on context
   — venue, match stage and season all influence predictions.
   This is the practical value of the system for team management.

4. Who the top performers are at specific venues and seasons
   — directly actionable for team selection decisions.

5. What any individual player is predicted to score or concede
   — the most user-facing feature of the entire project.

The intended audience is cricket analysts, IPL franchise
management and coaches who need pre-match performance estimates
to inform team selection, batting order and bowling strategy.
They are not data scientists so the dashboard must communicate
findings clearly without requiring any technical knowledge.

================================================================
MAIN STORY AND MESSAGE
================================================================

The main story this dashboard communicates is:

"IPL batting performance can be predicted with high accuracy
using machine learning and historical match data. Random Forest
achieves R² of 0.95 for batting prediction. Bowling economy
is harder to predict because match-day factors like pitch
conditions and opposition form on match day are not captured
in historical data alone. The interactive prediction system
allows anyone to generate pre-match performance estimates for
any player at any venue and match stage."

The key message for the audience is:
- Trust the batting predictions — they are accurate and
  validated against real player reputations
- Use bowling predictions as a guide, not a guarantee —
  economy rate is inherently variable
- The context matters — the same player will be predicted
  differently at different venues and stages, which reflects
  real cricket patterns

================================================================
DESIGN THINKING PROCESS
================================================================

Before building the dashboard I sketched the layout on paper
to decide what information belongs where and how a user would
naturally navigate through the findings.

The key design questions I asked were:
- What does a cricket analyst need to see first?
- How do I make it clear which model to trust?
- How do I show rankings without overwhelming the user?
- How do I make predictions transparent and explainable?

The answers shaped the six-tab structure:

TAB 1 — MODEL PERFORMANCE (show this first)
The audience needs to immediately see that the models work
and which one is best. Showing all five models side by side
with the best one highlighted answers this instantly.

TAB 2 — FEATURE IMPORTANCE (explain why it works)
After seeing the scores, the natural question is "why does
it work?" Feature importance answers this by showing which
historical statistics the model relies on most.

TAB 3 — ACTUAL VS PREDICTED (prove accuracy)
A scatter plot where points cluster along the diagonal line
is the most intuitive proof of model accuracy for a
non-technical audience.

TAB 4 — STAGE ANALYSIS (add cricket context)
Showing that average performance differs across league,
playoffs and finals grounds the model in cricket reality
and justifies why stage is included as a feature.

TAB 5 — PLAYER RANKINGS (make it explorable)
Allowing the user to filter by venue, season and match type
and see ranked players makes the data immediately useful
for real team selection decisions.

TAB 6 — PLAYER PREDICTION (make it actionable)
The prediction tool is the most practical output of the
entire project. Showing base prediction plus venue
adjustment plus stage adjustment makes the output
transparent and trustworthy rather than a black box.

================================================================
DESIGN PRINCIPLES APPLIED
================================================================

1. Colour consistency
   Blue is used for batting and green for bowling throughout
   all six tabs. The audience never needs to read a legend
   to know which sport they are looking at.

2. Highlight the best
   The best performing model is shown in a darker shade on
   comparison charts. The key finding is immediately visible
   without reading every number.

3. Values on bars
   Every bar chart shows the exact value on top of each bar
   so the audience gets both the visual pattern and the
   precise number at the same time.

4. Rank numbers inside bars
   Player Rankings show #1 #2 #3 labels inside the bars
   so the ranking order is unambiguous even for users who
   scan quickly.

5. Match counts in brackets
   Rankings show how many matches each result is based on.
   For example 32.5 (14m) means 14 matches. This lets the
   audience judge whether a ranking is based on enough data
   to be reliable.

6. Transparent prediction breakdown
   The prediction output shows base prediction plus venue
   adjustment plus stage adjustment separately. The audience
   understands HOW the number was reached, not just what it
   is. This makes the system trustworthy rather than a
   black box.

7. Spacing and breathing room
   Section labels, consistent margins and grouped controls
   make each tab feel organised and easy to scan rather
   than cluttered.

================================================================
WHAT I LEARNED FROM THIS MODULE
================================================================

The most important insight from this module is that a good
visualization is not about making something look attractive.
Every design decision should serve the audience's understanding.

The decision to show the prediction breakdown rather than just
the final number was the most important design choice in this
project. It transforms the prediction from a black box into
an explainable, trustworthy result that a cricket analyst
can actually act on.

I also learned that interactive tools are more appropriate
than static charts when the audience needs to explore data
themselves. A coach asking about a specific player at a
specific venue cannot get that answer from a static chart
but can get it instantly from the prediction widget.

The professor feedback on spacing and adding player rankings
with filters was valuable. It confirmed that usability
matters as much as accuracy when communicating to a
non-technical audience.

================================================================
DECISION: DASHBOARD OVER RESEARCH PAPER
================================================================

I chose the dashboard pathway because:

1. The project produces interactive predictions that are
   better demonstrated through a live tool than a static paper.

2. The intended audience (cricket analysts and team management)
   benefits more from an exploratory tool than from reading
   a methods section.

3. The prediction widgets naturally fit the dashboard format
   and would be lost in a paper format.

4. A dashboard communicates the practical value of the models
   more effectively for decision-making purposes.
