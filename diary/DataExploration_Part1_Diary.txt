DIARY ENTRY: DATA EXPLORATION — PART I (DATA PREVIEW)

At this stage, I constructed a focused analysis dataset by selecting only the tables and variables directly relevant to the project objective: studying how competitive pressure (league, playoffs, finals) relates to performance risk and variability in IPL players.

Only player–match level records were retained, as this is the appropriate granularity for analyzing performance variability across match stages.

⸻

FEATURES RETAINED

For batting analysis, the following variables were kept:
- match_id (link to match context)
- player
- runs
- balls
- strike_rate
- low_score indicator (runs < 10)
- stage_bucket (league / playoffs / final)
- season, date, venue (contextual variables)

For bowling analysis, the following variables were kept:
- match_id
- bowler
- overs
- runs_conceded
- wickets
- economy
- dot_ball_rate
- strike_rate_balls_per_wicket
- stage_bucket
- season, date, venue

Unnecessary columns unrelated to performance risk were excluded to maintain clarity and interpretability.

⸻

DESCRIPTIVE PREVIEW

The datasets were inspected using:
- .head()
- .info()
- .describe()

These checks verified:
- Correct data types (numeric vs categorical)
- No structural inconsistencies
- Reasonable ranges for performance metrics

Summary statistics confirmed expected variability in runs, economy, and wickets across match stages.

⸻

MISSINGNESS & STRUCTURAL OBSERVATIONS

Missing values were quantified using percentage-based checks.

Most missing values were structural rather than errors. For example:
- Bowling strike rate is undefined when a bowler takes zero wickets.
- Some contextual match fields contain occasional null values.

No data corrections were performed at this stage; observations were documented for transparency.

⸻

STAGE DISTRIBUTION

Stage distribution counts revealed strong imbalance:
- League matches dominate the dataset.
- Playoffs are less frequent.
- Finals represent the smallest subset.

This imbalance was not corrected but will be considered during interpretation and modeling.

⸻

SUMMARY

This initial exploration confirms that the dataset is structurally sound, stage-aware, and suitable for constructing pressure-based performance risk metrics. The cleaned player–match level dataset now serves as the foundation for risk index construction and future machine learning modeling.
