DIARY ENTRY: DATA ACQUISITION II — DATA STORAGE AND DATABASE SCHEMA

PROJECT CONTEXT:
This project integrates structured cricket performance data with unstructured fan sentiment data to analyze whether external pressure is associated with increased variability and risk in IPL player performance. To ensure reproducibility and clarity, all data sources are organized into a relational SQL database.

--------------------------------------------------

DATABASE SYSTEM:
A relational SQLite database is used because the project contains multiple related entities (matches, players, performances, videos, and comments) that require explicit relationships and structured joins.

--------------------------------------------------

STRUCTURED TABLES (CRICKET DATA):

TABLE: matches
Primary Key: match_id
Fields:
- match_date
- season
- venue
- match_type
- stage_bucket (league / playoffs / final)

This table stores match-level metadata and provides the temporal and contextual backbone of the analysis.

TABLE: players
Primary Key: player_name
This reference table standardizes player identities.

TABLE: player_match_batting
Primary Key: pm_id
Foreign Keys:
- match_id → matches.match_id
- player_name → players.player_name

Fields:
- runs
- balls
- out
- strike_rate

This table stores player-level performance outcomes per match and is the primary source for performance variability analysis.

--------------------------------------------------

UNSTRUCTURED MEDIA TABLES (YOUTUBE DATA):

TABLE: youtube_videos
Primary Key: video_id
Fields:
- title
- channel
- published_at
- query_used
- pressure_score
- stage_inferred
- team_mentioned
- match_date_inferred

This table stores metadata for YouTube videos used to collect fan reactions.

TABLE: youtube_comments
Primary Key: comment_id
Foreign Key:
- video_id → youtube_videos.video_id

Fields:
- comment_text
- like_count
- comment_published_at
- stage
- sentiment
- sentiment_bin

This table stores individual fan comments along with NLP-derived sentiment features.

--------------------------------------------------

MATCH–MEDIA LINKING TABLE:

TABLE: match_media_links
Primary Key: link_id
Foreign Keys:
- match_id → matches.match_id
- video_id → youtube_videos.video_id

Fields:
- link_method (date window, keyword match)
- link_confidence (0–1 score)
- notes

This table explicitly links YouTube media to cricket matches, allowing fan sentiment to be aligned with player performance. This resolves the integration gap between unstructured and structured data sources.

--------------------------------------------------

HANDLING UNSTRUCTURED DATA:
Raw comment text is stored alongside sentiment scores to avoid repeated NLP computation. Aggregate pressure indicators will be derived via SQL queries or feature engineering pipelines.

--------------------------------------------------

SUMMARY:
This database schema integrates cricket performance data and fan sentiment into a single relational structure that supports exploratory analysis, interpretable modeling, and future machine learning extensions.
