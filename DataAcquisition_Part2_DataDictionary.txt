DATA ACQUISITION PART II — DATA DICTIONARY (Version 1)

Project: Quantifying Performance Risk in Professional Cricket Across Match Stages

This data dictionary defines the core variables stored in the relational database for Milestone 1.
All data is derived exclusively from structured cricket match and player performance records.

The goal of this database is to support analysis of performance variability and risk for both batters and bowlers under different match pressure stages (league, playoffs, finals).

⸻

TABLE: matches

id (INTEGER, Primary Key)
	•	Unique identifier for each IPL match.

date (DATE)
	•	Date on which the match was played.

season (INTEGER)
	•	Season/year extracted from the match date.

venue (TEXT)
	•	Stadium or city where the match was played.

team1 (TEXT)
	•	First competing team.

team2 (TEXT)
	•	Second competing team.

match_type (TEXT)
	•	Match format (league match, qualifier, eliminator, final).

stage_bucket (TEXT)
	•	Derived match pressure category:
	•	league
	•	playoffs
	•	final

winner (TEXT)
	•	Winning team.

result (TEXT)
	•	Match outcome type (runs, wickets, tie, no result).

⸻

TABLE: players

player_name (TEXT, Primary Key)
	•	Standardized player name used consistently across all performance records.

This table serves as a reference table for all player roles, including:
	•	Specialist batters
	•	Specialist bowlers
	•	All-rounders

⸻

TABLE: player_match

This table stores player–match level performance outcomes for both batting and bowling roles.

match_id (INTEGER, Foreign Key → matches.id)
	•	Match identifier linking performance to match context.

player (TEXT, Foreign Key → players.player_name)
	•	Player involved in the match.

Batting Performance Fields

runs (INTEGER)
	•	Runs scored by the player in the match.

balls (INTEGER)
	•	Balls faced by the player.

out (INTEGER: 0/1)
	•	1 if dismissed, 0 otherwise.

strike_rate (REAL)
	•	Batting strike rate: (runs / balls) × 100.

low_score (INTEGER: 0/1)
	•	Derived indicator:
	•	1 if runs < 10
	•	0 otherwise
	•	Used as a proxy for batting failure risk.

Bowling Performance Fields (Current or Extendable)

(Some may already exist or will be derived in later milestones)

overs_bowled (REAL)
	•	Number of overs bowled by the player.

runs_conceded (INTEGER)
	•	Runs conceded while bowling.

wickets (INTEGER)
	•	Number of wickets taken.

economy_rate (REAL)
	•	Runs conceded per over.

bowling_impact_flag (INTEGER: 0/1) (derived)
	•	Indicator for high-impact bowling performances (e.g., multiple wickets or low economy).

Context Fields

date (DATE)
	•	Match date (duplicated for convenience).

season (INTEGER)
	•	Season/year of the match.

venue (TEXT)
	•	Match venue.

stage_bucket (TEXT)
	•	Match pressure category inherited from matches.

⸻

DERIVED ANALYSIS FEATURES (NOT STORED AS RAW TABLES)

Computed during EDA and modeling:

Batting Risk Metrics
	•	avg_runs
	•	low_score_rate
	•	run_variance
	•	pressure_impact_index (batting)

Bowling Risk Metrics
	•	avg_economy
	•	wicket_variance
	•	high_concession_rate
	•	pressure_impact_index (bowling)

Labels

pressure_label
	•	Stable
	•	More risky under pressure
	•	High pressure drop

⸻

NOTES AND SCOPE
	•	Milestone 1 focuses on:
	•	Data acquisition
	•	Database design
	•	Exploration readiness
	•	Both batting and bowling roles are supported at the schema level.
	•	All variables are derived from objective match outcomes, not psychological inference.
	•	Later milestones may:
	•	Separate batting vs bowling models
	•	Introduce role-specific pressure metrics
	•	Extend to team-level pressure dynamics

⸻

SUMMARY

This data dictionary defines a cricket-only, role-aware relational schema that supports exploratory analysis of performance risk for both batters and bowlers across match stages.
It provides a defensible, reproducible foundation for advanced analytics and machine learning without relying on external or subjective data sources.
