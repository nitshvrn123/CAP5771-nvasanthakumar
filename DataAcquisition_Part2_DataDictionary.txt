DATA ACQUISITION PART II — DATA DICTIONARY (Version 1)

Project: Quantifying Performance Risk in Professional Cricket Across Match Stages

This data dictionary defines the core variables stored in the relational database for Milestone 1.
All data is derived exclusively from structured cricket match and player performance records.

The goal of this database is to support analysis of performance variability and risk under different match pressure stages (league, playoffs, finals).

⸻

TABLE: matches

id (INTEGER, Primary Key)
	•	Unique identifier for each IPL match.

date (DATE)
	•	Date on which the match was played.

season (INTEGER)
	•	Season/year extracted from the match date.

venue (TEXT)
	•	Stadium or city where the match was played.

team1 (TEXT)
	•	First competing team.

team2 (TEXT)
	•	Second competing team.

match_type (TEXT)
	•	Match format (e.g., league match, qualifier, eliminator, final).

stage_bucket (TEXT)
	•	Derived match pressure category:
	•	league
	•	playoffs
	•	final

winner (TEXT)
	•	Winning team for the match.

result (TEXT)
	•	Match outcome type (runs, wickets, tie, no result).

⸻

TABLE: players

player_name (TEXT, Primary Key)
	•	Standardized player name used consistently across all performance records.

This table serves as a reference table to ensure consistent player identity across matches.

⸻

TABLE: player_match

match_id (INTEGER, Foreign Key → matches.id)
	•	Match identifier linking performance to match context.

player (TEXT, Foreign Key → players.player_name)
	•	Player involved in the match.

runs (INTEGER)
	•	Runs scored by the player in the match.

balls (INTEGER)
	•	Number of balls faced.

out (INTEGER: 0/1)
	•	Indicator of dismissal:
	•	1 = dismissed
	•	0 = not dismissed

strike_rate (REAL)
	•	Batting strike rate calculated as (runs / balls) × 100.

low_score (INTEGER: 0/1)
	•	Derived indicator:
	•	1 if runs < 10
	•	0 otherwise
Used as a proxy for performance failure or risk.

date (DATE)
	•	Match date (duplicated for convenience in analysis).

season (INTEGER)
	•	Season/year of the match.

venue (TEXT)
	•	Match venue.

stage_bucket (TEXT)
	•	Match pressure category inherited from the matches table.

⸻

DERIVED ANALYSIS FEATURES (NOT STORED AS RAW TABLES)

These variables are computed during analysis and EDA:

avg_runs
	•	Average runs scored by a player across matches.

low_score_rate
	•	Fraction of matches in which the player scored fewer than 10 runs.

performance_variance
	•	Variance of runs scored across matches.

pressure_impact_index
	•	Composite metric combining low-score probability and average performance under pressure-stage matches.

pressure_label
	•	Categorical interpretation of performance under pressure:
	•	Stable
	•	More risky under pressure
	•	High pressure drop

NOTES AND SCOPE
	•	This data dictionary reflects the Milestone 1 scope, which focuses on:
	•	Data acquisition
	•	Database design
	•	Exploration readiness
	•	All variables are derived from objective match outcomes, not subjective or psychological inference.
	•	Future milestones may extend this dictionary to include bowling metrics, role-based features, or predictive targets.

⸻

SUMMARY

This data dictionary defines a clean, cricket-only relational schema that supports exploratory analysis of performance risk across match stages. It provides a reproducible foundation for statistical analysis and machine learning modeling without relying on external or subjective data sources.
